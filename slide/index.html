进程间传递共享内存


                        class SharedMemory:
                            @classmethod
                            def create(cls, size):
                                fd = libc.memfd_create('overmind', os.O_RDWR)
                                os.ftruncate(fd, size)
                                return cls(fd=fd)

                            # Pass pid & fd to client

                            @classmethod
                            def rebuild(cls, pid, fd):
                                fd = os.open(f'/proc/{pid}/fd/{fd}', os.O_RDWR)
                                return cls(fd=fd)

绕过了 POSIX shm

不用纠结 SCM_RIGHTS 那一坨难看的 API

解释下 pid & fd 的跨进程 race condition。 扩展下 linux 下的各种奇怪的 fd 机制，signalfd, eventfd, timerfd, pidfd

你这古法手工的轮子，它圆吗?

  • POSIX shm 容量有限（docker 默认给 64M）
  • torch 会给每个 UntypedStorage 申请一个 POSIX shm，即使只有几个字节
  • torch 会给每个 shm 做引用计数，pickle 时增加，unpickle 时减少，不能复用 pickle 结果
  • 每个 shm 会占用一个 fd
