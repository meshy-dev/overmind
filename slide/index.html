如何工作

(Oversimplified)

  • 把函数和参数都 pickle 起来，发一个 RPC 给 overmind daemon
  • 如果没有 overmind daemon 就拉起来一个
  • 在 overmind 的进程中执行发过来的函数
  • 把结果再 pickle 一下返回给 client

如何工作 (cont.)

  • 把函数和参数都 pickle 起来，发一个 RPC 给 overmind daemon
  • 发现同样的函数和参数之前调用过了
  • 直接把之前的 pickle 返回！

  • 为了支持一些复杂的情况（主要是 closure），被迫用了 dill 作为 pickler 的基座，纯 python 实现速度很慢
  • 因为是一次性的开销所以没有努力优化

Pickler 里都加了哪些料？

Pickler 里都加了哪些料？

  • torch.multiprocessing.reductions
  • 自定义 torch.UntypedStorage 的 pickle 逻辑

UntypedStorage reduce & rebuild


                        def _reduce(storage: torch.UntypedStorage):
                            device = storage.device
                            storage = storage.cpu()
                            frag = hoarder.put(storage)
                            return (_rebuild, (frag, device))

                        def _rebuild(frag, device) -> torch.UntypedStorage:
                            mv: memoryview = borrower.borrow(frag)
                            storage: UntypedStorage = _make_untyped_storage(mv)
                            if device.type == 'cpu':
                                return storage
                            elif device.type == 'cuda':
                                return storage.cuda(device)

进程间传递共享内存


                        class SharedMemory:
                            @classmethod
                            def create(cls, size):
                                fd = libc.memfd_create('overmind', os.O_RDWR)
                                os.ftruncate(fd, size)
                                return cls(fd=fd)

                            # Pass pid & fd to client

                            @classmethod
                            def rebuild(cls, pid, fd):
                                fd = os.open(f'/proc/{pid}/fd/{fd}', os.O_RDWR)
                                return cls(fd=fd)

绕过了 POSIX shm

解释下 pid & fd 的 race condition

Why reinventing the wheel?

  • POSIX shm 容量有限（docker 默认给 64M）
  • torch 会给每个 UntypedStorage 申请一个 POSIX shm，即使只有几个字节
  • torch 会给每个 shm 做引用计数，pickle 时增加，unpickle 时减少，不能复用 pickle 结果
  • 每个 shm 会占用一个 fd
